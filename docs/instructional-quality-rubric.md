# Instructional Quality Rubric

Use this rubric when adding or reviewing CodeHerWay lessons and quizzes. It is
designed to keep the platform calm and practical while still making learning
evidence visible.

## Lesson Signals

Each lesson should include most of these signals:

- Objective: the learner knows what capability they are building.
- Misconception check: the lesson names a likely mistake or confusion point.
- Retrieval prompt: the learner must explain or recall something without copying.
- Guided practice: the learner sees a model, steps, starter code, or worked example.
- Independent practice: the learner has a small task beyond reading.
- Transfer: the lesson points toward a project, portfolio use, or the next concept.

The content audit reports thin lessons as warnings, not blockers, so existing
curriculum can improve incrementally.

## Quiz Signals

Quizzes should include more than recognition questions. A strong quiz includes:

- Misconception coverage: one item catches a common mistake.
- Reasoning coverage: one item asks the learner to interpret, predict, debug, or explain.
- Application coverage: one item uses a realistic scenario or code decision.

The goal is not harder quizzes for their own sake. The goal is feedback that
helps a beginner understand what to review and what to practice next.
